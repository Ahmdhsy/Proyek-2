Dynamic Routing
OSPF (Open Shortest Path First)merupakan salah satu protokol yang diklasifikasikan kedalam jenis protokol dynamic routing. Dynamic sendiri diciptakan untuk mengatasi kelemahan kelemahan yang ada pada static routing. Kelemahan - kelemahan tersebut diantaranya adalah
1.Static routing hanya bisa digunakan untuk jaringan berskala kecil dan stabil. Hal ini dikarenakan static routing di maintenance dan di konfigurasi secara manual. Sehingga static routing tidak bisa beradaptasi pada jaringan berskala besar

2.Static routing tidak dapat secara otomatis menyesuaikan diri dengan perubahan yang terjadi pada jaringan. Hal ini disebabkan karena routing static dikonfigurasi secara manual, dan konfigurasi ini tidak akan berubah secara otomatis ketika terjadi perubahan pada jaringan.

Atas dasar kelemahan-kelemahan tersebut dynamic routing diciptakan. Dynamic routing sendiri terbagi menjadi 2 yaitu berdasarkan Ass dan berdasarkan mekanisme kerja dan algoritma.

Dynemic routing berdasarkan Ass terbagi menjadi 2 yaitu:
1.Interior Gateway Protocol (IGPs)
Protokol yang termasuk ke dalam jenis ini adalah OSPF, RIP, dan IS-IS.

2.Exterior Gateway Protocol (EGPs)
Protokol yang termasuk ke dalam jenis ini adalah BGIP

Sedangkan Dynemic routing berdasarkan mekanisme kerja dan algoritma juga terbagi menjadi 2 yang mencakup:
1.Distance Vector Routing Protocol
Protokol yang termasuk ke dalam jenis ini adalah RIP

2.Link State Routing Protocol 
Protokol yang termasuk ke dalam jenis ini adalah OSPF, dan IS-IS.

OSPF
OSPF (Open Shortest Path First) merupakan salah satu protokol routing yang menggunakan jenis link state routing.Dalam jenis routing ini, setiap router akan menghasilkan LSA (Link State Advertisements) yang berisi informasi interface Router. Selain LSA, router juga akan membuat database yang disebut link state database (LSDB) dan akan mendaftarkan LSA yang diterima ke dalam LSDB yang telah dibuat. Didalam LSDB sendiri akan tersimpan beberapa informasi yang dibawa oleh LSA yaitu interface router dan topologi jaringan. Selain pertukaran informasi dan pembentukan database, dalam OSPF juga terjadi proses perhitungan SPF. Dengan menggunakan algoritma SPF, OSPF dapat menemukan jalur terbaik untuk mengirimkan paket data di jaringan, sehingga membuat pengiriman data menjadi lebih efisien dan jaringan dapat beradaptasi dengan perubahan topologi dengan cepat. Dalam perhitungan SPF setiap router akan melakukan pohon loop-free (bebas loop) untuk memastikan bahwa tidak ada loop routing yang terbentuk di jaringan dan menentukan jalur optimal. Pada tahap akhir setiap rute akan mendaftarkan rute hasil dari perhitungan SPF tadi ke dalam tabel routing setiap router.

OSPF Area
Didalam OSPF terdapat istilah yang disebut OSPF area yang digunakan untuk mengidentifikasi area OSPF. dalam area ini setiap router akan diidentifikasi menggunakan Area ID. setelah dilakukan identitfikasi menggunakan area ID

OSPF Router ID
Setelah melakukan proses identifikasi menggunakan area menggunakan area ID, selanjutnya setiap router yang ada pada area ID akan dilakukan proses identifikasi menggunakan router ID. Router ID sendiri dapat dilakukan konfigurasi secara manual maupun dikonfigurasi secara otomatis oleh sistem. 

OSPF Cost Value
Selain area dan router ID, dalam protokol routing OSPF (Open Shortest Path First), dikenal dengan istilah cost value (nilai cost), cost value adalah metrik yang digunakan untuk menentukan jalur terbaik (paling efisien) untuk mengirimkan paket data di jaringan. Router OSPF menggunakan cost value untuk menghitung biaya kumulatif yang terkait dengan setiap jalur yang tersedia. Semakin rendah cost value, maka semakin disukai jalur tersebut oleh OSPF.

Jenis Paket OSPF
Terdapat 5 jenis paket dalam OSPF yang digunakan untuk interaksi antara router.
1.Paket Hello
Digunakan untuk maintenance antar setiap router. dalam paket ini mengandung informasi seperti alamat IP router, ID area OSPF, dan cost link.
2.Paket Deskripsi Database
Digunakan untuk menukar informasi database LSA (Link State Advertisement) antar router. dalam paket ini mengandung informasi seperti nomor sequence LSA, checksum, dan LSA types yang dimiliki router.
3.Paket Link State Request
Digunakan untuk meminta LSA spesifik dari router lain. Paket ini akan digunakan dan dikirim ke router lain ketika router membutuhkan informasi LSA yang belum dimilikinya.
4.Paket Link State Update
Digunakan untuk mengirim update LSA ke semua router OSPF di area yang sama. paket ini akan digunakan dan dikirim ke router lain ketika terjadi perubahan topologi jaringan, seperti link yang up atau down.
5.Link State ACK
Digunakan untuk mengkonfirmasi penerimaan paket OSPF. Paket ini bertujuan untuk memastikan bahwa paket OSPF diterima dengan benar dan tidak ada data yang hilang.

Tiga jenis entry OSPF
Didalam OSPF, terdapat tiga tabel yang dapat dilakukan proses entry yaitu tabel neighbor OSPF, LSDB, dan routing OSPF

1.Tabel neighbor OSPF
Tabel ini berfungsi untuk menjelaskan status neighbor atau relationship antara router OSPF, untuk dapat melihat tabel ini kita dapat menggunakan printah "display ospf peer" pada comand line router di ensp. di dalam tabel ini juga berisi informasi seperti alamat interface dan router id dari router OSPF lain yang memiliki hubungan.

2.Tabel LSDB
Tabel LSDB ini sebagaimana yang telah disebutkan sebelumnya digunakan untuk menyimpan informasi berupa LSA dari router sendiri dan router ospf lain yang memiliki hubungan neighbor. Dalam tabel ini berisi informasi seperti tipe perangkat, Link State ID, AdvRouter, Age, dan metric. Untuk dapat melihat tabel ini kita dapat menggunakan printah "display ospf lsdb" pada comand line router di ensp.

3.Tabel Routing OSPF
Tabel Routing OSPF digunakan untuk menyimpan informasi IP tujuan (destination), cost, dan alamat ip hop. untuk dapat melihat tabel ini kita dapat menggunakan printah "display ospf routing" pada comand line router di ensp.
